# 2. Attacker Methodology Introduction

## Overview
Cyber defense requires understanding both:
- **Enterprise network architecture**
- **Attacker mindset and methodology**

Attackers follow structured approaches to exploit vulnerabilities. Defenders must anticipate these actions and design **defense-in-depth strategies** accordingly.

Key frameworks:
- **Lockheed Martin Cyber Kill Chain → "WHY" attacks happen (phases)**
- **MITRE ATT&CK → "HOW" attacks are executed (techniques)**

---

## 2.1 The Network as a Whole

### Key Concepts
- Enterprise networks are **complex and unique**
- Core principle: **Defense-in-Depth (DiD)**
  - Multiple layered controls
  - Redundancy if one layer fails

---

### 2.1.1 DMZ (Demilitarized Zone)
- Hosts **internet-facing services** (e.g., web, email)
- Acts as a **buffer between internal network and internet**
- Typical setup:
  - Frontend firewall → filters incoming traffic
  - Backend firewall → protects internal network
- **High-risk zone** → common entry point for attackers

---

### 2.1.2 Deployment Environments
Separates development lifecycle to reduce risk:

| Environment | Purpose |
|------------|--------|
| Dev        | Development & testing |
| Staging    | Production-like testing |
| Production | Live system |

Key points:
- Prevents unstable code from affecting production
- Enables **security testing before release**
- Compromise in dev ≠ direct impact, but still risky

---

### 2.1.3 Core vs Edge Devices

| Type | Role |
|------|------|
| Edge Devices | Connect networks (e.g., internet ↔ internal) |
| Core Devices | High-speed internal routing |

- Edge = entry/exit point (higher exposure)
- Core = performance + routing
- **Challenge:** tracing attack origin across layers
- Requires logs, packet capture, SIEM

---

### 2.1.4 VPN & Remote Sites
- Enables **secure remote access**
- Extends enterprise network across locations

Risks:
- Weak configuration → attacker entry point
- Compromised remote site = full network risk

---

## 2.2 Lockheed Martin Cyber Kill Chain

### Purpose
Describes **stages of an attack lifecycle**

### Phases
1. Reconnaissance
2. Weaponization
3. Delivery
4. Exploitation
5. Installation
6. Command & Control (C2)
7. Actions on Objectives

### Key Insight
- Each phase depends on the previous one
- **Breaking any phase disrupts the attack**

---

### Case Study: Monero Cryptomining
- Exploited vulnerabilities in:
  - Jenkins
  - Oracle WebLogic

Common indicators:
- PowerShell payload
- Same C2 IP
- Same malware file

➡️ **Insight:** Reused techniques help detection & attribution

---

### Case Study: Petya → Mischa → GoldenEye

Evolution of ransomware:

| Variant | Key Behavior |
|--------|-------------|
| Petya  | Encrypts MBR + MFT (system unusable) |
| Mischa | Encrypts user files (fallback if no admin rights) |
| GoldenEye | Combines both + privilege escalation |

➡️ Attackers **adapt based on detection feedback**

---

## 2.3 MITRE ATT&CK Framework

### Purpose
Describes **technical execution of attacks**

- Focus: **Tactics, Techniques, Sub-techniques (TTPs)**
- Used for:
  - Detection engineering
  - Threat hunting
  - Standardized reporting

---

### ATT&CK Tactics (High-Level Goals)
- Reconnaissance
- Initial Access
- Execution
- Persistence
- Privilege Escalation
- Defense Evasion
- Credential Access
- Discovery
- Lateral Movement
- Collection
- Command & Control
- Exfiltration
- Impact

---

### Structure

| Level | Description |
|------|------------|
| Tactic | Goal (WHY) |
| Technique | Method (HOW) |
| Sub-technique | Specific implementation |

Example:
- Privilege Escalation
  - Abuse Elevation Control Mechanism
    - Bypass UAC

---

### Case Study: OilRig (APT34)
- Initial Access → Spearphishing
- Execution → Malicious Office macros / PowerShell
- C2 → Web + DNS protocols

➡️ Uses both **custom tools + native OS tools**

---

### Case Study: APT3
- Uses **zero-day vulnerabilities**
- Persistence:
  - Local accounts
  - Scheduled tasks
- Privilege Escalation:
  - SYSTEM-level execution
  - Sticky Keys abuse

---

### Case Study: APT28 (Fancy Bear)
- Lateral Movement:
  - EternalBlue (SMB exploit)
- Credential Access:
  - NetBIOS poisoning (Responder)
- C2:
  - WebSocket-based communication

➡️ Uses mix of:
- Custom malware
- Open-source tools
- Native protocols

---

## Key Takeaways

- **Defense requires attacker mindset**
- **Defense-in-depth is critical**
- Kill Chain = attack phases (**WHY**)
- MITRE ATT&CK = attack techniques (**HOW**)
- Attackers:
  - Reuse techniques
  - Adapt quickly
  - Learn from defenders

➡️ Effective defense = combining:
- Network understanding
- Kill Chain disruption
- ATT&CK-based detection
